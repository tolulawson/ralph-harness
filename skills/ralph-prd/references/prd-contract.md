# Ralph PRD Contract

Use a structured PRD that includes:

1. title and summary
2. problem statement
3. goals
4. epoch map
5. target users
6. user stories
7. functional requirements
8. non-goals
9. success metrics
10. open questions or assumptions

Quality bar:

- ask only `3-5` clarifying questions when needed
- keep stories small
- keep language accessible to a junior developer or coding agent
- write acceptance criteria that can be verified later
- define epochs as planning groups, not execution units
- do not implement during the PRD step
